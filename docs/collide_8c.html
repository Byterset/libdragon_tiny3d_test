Attempts to create a new contact in the collision_scene and add it to the physics_objects list of active contacts.

It is important to know in which order the objects were tested during the EPA so the correct contact point of the result gets added.

Corrects the velocities of colliding objects. This can affect both the linear and angular velocities of an object depending on if it has any constraints or no rotation reference.

This will also correct velocities for collisions against static mesh_colliders. Here one of the input object pointers will be NULL and act as a collision against an object with infinite mass.

Parameters
apointer to colliding physics_object a
bpointer to colliding physics_object b
resultthe result of the EPA, containing collision normal, contact points & penetration depth
frictionthe combined friction of the objects
bouncethe combined bounce of the objects
